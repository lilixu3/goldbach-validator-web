# 哥德巴赫猜想验证器设计说明

## 目标
- 提供一个网页端静态验证器。
- 用户输入 **大于 6 的偶数** 后，点击按钮开始验证。
- 页面支持 **分批计算、分页查看、页码跳转、进度反馈、手动停止**。
- 页面显示该数字对应的 **全部素数对**，并展示 **总组数**。

## 已确认需求
- 页面风格：保留 **A｜极简数学工具** 基调，但整体 **更简洁、文字更少**。
- 触发方式：**点击按钮**。
- 结果呈现：**列出全部素数对 + 显示总组数**。
- 分页容量：**50 组 / 页**。
- 分页样式：**B｜紧凑跳转**。
- 大数字处理：**不设输入上限，改为分批计算**。
- 长计算反馈：**进度条 + 停止按钮**。

## 页面结构
1. 顶部区：
   - 仅保留简短标题 `哥德巴赫验证器`。
   - 可保留一条极短副文案，不做大段说明。
2. 输入区：
   - 一个数字输入框。
   - 一个 `开始验证` 按钮。
3. 状态区：
   - `已找到 X 组`
   - 进度条与百分比
   - `停止` 按钮
4. 结果区：
   - 仅渲染当前页的 50 组结果
   - 结果卡片风格继续保持简洁
5. 分页区：
   - `上一页 / 下一页`
   - `当前页 / 总页数`
   - 页码输入框
   - `跳转` 按钮

## 交互规则
- 仅接受整数输入。
- 输入为空时提示“请输入一个数字。”
- 输入不是整数时提示“请输入整数。”
- 输入不是大于 6 的偶数时提示“请输入大于 6 的偶数。”
- 用户点击 `开始验证` 时：
   - 若已有旧计算在进行，先终止旧计算，再开启新计算。
   - 页面重置到第 1 页。
   - 清空旧结果，开始分批生成新结果。
- 计算过程中：
   - 已找到的结果实时累加。
   - 进度条和百分比实时更新。
   - 用户可点击 `停止` 立即终止本轮计算。
   - 停止后保留当前已找到结果，允许继续翻页查看。
- 分页规则：
   - 每页固定 50 组。
   - `上一页 / 下一页` 只在可用时启用。
   - 跳转仅允许进入已生成的页；若目标页尚未生成，给出极短提示，如“该页未生成”。
   - 当计算完成后，分页总数固定为最终总页数。

## 算法与性能策略
- 主循环仍从 `2` 遍历到 `n / 2`。
- 若 `p` 与 `n - p` 同时为素数，则记录为一组。
- 只保留不重复组合，因此不会同时出现 `5 + 23` 与 `23 + 5`。
- 为避免大输入时界面长时间卡住，计算改为 **分批执行**：
   - 每批处理一段连续区间；
   - 每批结束后把控制权交还浏览器，再继续下一批；
   - 进度、已找到结果数、分页状态在批次间刷新。
- 渲染策略：
   - 不一次性渲染全部结果；
   - 始终只渲染当前页结果与必要的分页控件；
   - 计算中若当前页数据未变化，不重复重绘整页列表。

## 视觉方向
- 保持浅色纸面感，但进一步压缩文字和视觉层级。
- 标题、输入区、结果区都更紧凑，减少装饰信息。
- 分页与进度模块统一成低干扰、紧凑型控件。
- 保留细边框、柔和阴影与少量动效，避免厚重卡片和冗长说明。

## 验证范围
- 基础合法示例：`8`、`10`、`28`、`100`
- 大输入示例：选择明显更大的偶数，确认页面仍可操作
- 非法示例：空输入、`6`、`7`、`3.5`、`abc`
- 分页验证：
   - 超过 50 组时正确翻页
   - 页码跳转正确
   - 计算中尝试跳到未生成页时提示正确
- 进度验证：
   - 计算中进度会变化
   - 点击 `停止` 后能立即停止后续批次
